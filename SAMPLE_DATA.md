## How to Use This Data

1. **Testing System Accuracy**: Use these JDs to test the resume parsing and matching system
2. **Skill Dictionary Calibration**: Verify that the skill extraction works accurately across diverse job definitions
3. **Experience Extraction**: Test date range and years calculation against different formatting styles
4. **Real-World Scenarios**: These are actual job descriptions with real skill requirements and salary ranges

## Integration with Resume Parsing

The system parses each JD and extracts:
- ✅ **Salary**: Handles USD, LPA, INR formats
- ✅ **Experience Requirements**: Extracts years as numeric range or fresher status
- ✅ **Required Skills**: High-precision skill matching against ~300+ tech skills dictionary
- ✅ **Optional Skills**: Distinguishes between required and nice-to-have skills
- ✅ **Role Summary**: Extracts position title and role description

## Testing Recommendations

1. **Test each JD** with the sample resume to validate skill matching accuracy
2. **Compare scores** across different job types to ensure consistency
3. **Verify salary parsing** for edge cases (ranges, different currencies)
4. **Validate experience extraction** against various date formats
5. **Check skill deduplication** when the same skill appears in multiple sections
